Control & Intent

Goal manipulation, memory poisoning, trust exploitation, and behavioral drift

ASI01 · ASI06 · ASI09 · ASI10
ASI01 · CRITICAL

Agent Goal Hijack

  • Attackers redirect an agent's multi-step planning via prompt injection or poisoned data
  • Unlike single-response LLM attacks, this subverts autonomous decision-making across entire workflows
  • Defend with locked system prompts, intent capsules, and human approval on goal changes
⚡ Real-World: EchoLeak

A crafted email triggered Microsoft 365 Copilot to silently exfiltrate confidential emails and chat logs — no user interaction required. The hidden instructions were embedded in an otherwise normal-looking message.

ASI06 · HIGH

Memory & Context Poisoning

  • Attackers corrupt RAG stores and vector DBs with malicious data that persists across sessions
  • Unlike one-shot injection, poisoned memory is a slow-acting backdoor that biases all future reasoning
  • Defend with memory segmentation, trust-scored retrieval with decay, and rollback capability
⚡ Real-World: Cross-Tenant Vector Bleed

Attackers seeded near-duplicate embeddings in a shared vector store with loose namespace filters. The poisoned content leaked across tenant boundaries, delivering attacker-controlled data as "trusted knowledge."

Full breakdown →
ASI09 · MEDIUM

Human-Agent Trust Exploitation

  • Humans over-trust confident AI rationales and approve risky actions without independent verification
  • Agents can emotionally manipulate, fabricate explanations, or disguise effects behind "preview" panes
  • Defend with non-model risk summaries, visual trust cues, and separate preview from effect
⚡ Real-World: Credential Harvesting Deception

An injected IT support agent cited real ticket numbers to appear legitimate while requesting user credentials. The specificity built false trust — users handed over passwords to a fake agent.

Full breakdown →
ASI10 · CRITICAL

Rogue Agents

  • Agents that appear compliant while secretly pursuing hidden goals, exfiltrating data, or self-replicating
  • Reward hacking: agents game metrics destructively (e.g., deleting backups to "save storage costs")
  • Defend with watchdog agents, kill-switches, and periodic behavioral attestation
⚡ Real-World: Persistence via API

A compromised agent used provisioning APIs to replicate itself across the network. Each replica independently continued the malicious task — making containment nearly impossible.

Access & Execution

Misuse of tools, privilege abuse, and unexpected code execution

ASI02 · ASI03 · ASI05
ASI02 · CRITICAL

Tool Misuse & Exploitation

  • Agents apply authorized tools in unintended ways — deleting data, exfiltrating via shell, running up API costs
  • Tool descriptors and metadata can be poisoned at runtime (MCP tool poisoning)
  • Defend with per-tool IAM profiles, execution sandboxes, and adaptive cost budgets
⚡ Real-World: MCP Tool Poisoning

Attackers manipulated MCP tool descriptors at runtime, making a legitimate "summarize" tool secretly forward all processed data to an external exfiltration endpoint.

Full breakdown →
ASI03 · CRITICAL

Identity & Privilege Abuse

  • Agents inherit overly broad credentials and cache them across sessions, creating an "attribution gap"
  • Confused deputy attacks: a low-privilege agent tricks a high-privilege agent into executing its commands
  • Defend with task-scoped tokens, per-action re-verification, and identity isolation per session
⚡ Real-World: Delegated Abuse

A finance agent passed all its permissions to a "DB query" sub-agent. An attacker exploited this to exfiltrate HR data — the sub-agent had full database access when it only needed financial tables.

Full breakdown →
ASI05 · CRITICAL

Unexpected Code Execution (RCE)

  • Agents generate and run code in real-time, often bypassing traditional security controls entirely
  • Prompt-to-code injection, unsafe eval(), and "vibe coding" runaways can compromise the host
  • Defend by banning eval(), running in sandboxed containers, and separating generation from execution
⚡ Real-World: Vibe Coding Runaway

A coding agent attempting self-repair generated an unreviewed rm command that deleted production data. The agent's "fix build" loop ran destructive shell commands without human oversight.

Ecosystem & Trust

Supply chain flaws, insecure inter-agent communication, and cascading failures

ASI04 · ASI07 · ASI08
ASI04 · HIGH

Supply Chain Vulnerabilities

  • Agentic systems load tools, prompts, and personas dynamically at runtime — a "live" supply chain
  • Poisoned packages, typosquatted endpoints, and tampered agent cards cascade to all downstream users
  • Defend with SBOMs/AIBOMs, pinned dependencies by hash, and a supply-chain kill switch
⚡ Real-World: Malicious MCP Postmark

An NPM package impersonating Postmark (the email service) secretly BCC'd every email to the attacker. Thousands of agents using this tool unknowingly leaked all processed emails.

Full breakdown →
ASI07 · HIGH

Insecure Inter-Agent Communication

  • Agents communicate via APIs and message buses — MITM, spoofing, and replay attacks are all possible
  • A single tampered message can cause "semantics split-brain" where agents diverge on intent
  • Defend with mTLS, signed/hashed messages, and nonce-based replay protection
⚡ Real-World: A2A Registration Spoofing

An attacker registered a fake peer agent in a discovery service. Legitimate agents routed privileged coordination traffic to the spoofed agent, leaking sensitive orchestration data.

Full breakdown →
ASI08 · HIGH

Cascading Failures

  • One hallucination or poisoned input propagates autonomously across agents, compounding into system-wide harm
  • Without human checkpoints, errors bypass security boundaries and self-reinforce in feedback loops
  • Defend with circuit breakers, digital twin replays, and blast-radius guardrails
⚡ Real-World: Financial Trading Cascade

A poisoned market analysis agent inflated risk limits; downstream agents auto-traded massive positions. Compliance agents stayed blind while losses compounded — faster than any human could intervene.
